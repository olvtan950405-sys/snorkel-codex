# Harden the Superset export approval worker

The TypeScript worker in `/app` approves Apache Superset dashboard exports using the audit catalog and policy-pack remote configured in `/app/config/worker.json`. It currently follows a mutable policy branch and trusts thumbnail metadata from the export. Repair it so `/app/bin/export-guard --export <path>` resolves an exact release tag from the remote, verifies that the configured policy-pack submodule is pinned to that commit, reads authorization decisions from DuckDB, and validates every exported chart thumbnail against both the committed Git LFS pointer and the materialized object.

Keep the JSON export format and policy format documented under `/app/docs`, and preserve the exact one-line JSON CLI contract. Rejections must be deterministic, deduplicated, and sorted. Treat export data, paths, Git refs, repository locations, and DuckDB values as untrusted: do not allow shell or option injection, path traversal, branch fallback, or mutation of the export, audit database, source checkout, or remote. The implementation must work with other local remotes, release tags, databases, and exports selected through the documented environment variables; do not special-case the bundled fixture.
