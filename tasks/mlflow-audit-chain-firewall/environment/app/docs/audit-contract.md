# MLflow audit-ledger contract

Input is UTF-8 JSON Lines; blank lines are ignored. Each nonblank line is exactly an object with `seq` (integer from 0 through 9007199254740991), `method` (nonempty string), `path` (nonempty string), and optional `headers` (string-to-string object) and `body` (any JSON value). Unknown fields, duplicate keys at any depth, invalid JSON, non-integer or repeated `seq`, and decreasing sequence order are fatal. A fatal stream exits nonzero without creating or replacing the destination.

Normalize the method to uppercase. Split `path` at its first `?`, and remove exactly one trailing slash from the path portion except `/`. Match only these method/path pairs: `POST /api/2.0/mlflow/runs/log-metric` as `log_metric`, `POST /api/2.0/mlflow/runs/log-parameter` as `log_parameter`, `POST /api/2.0/mlflow/runs/set-tag` as `set_tag`, `GET /api/2.0/mlflow/runs/get` as `get_run`, and `DELETE /api/2.0/mlflow/runs/delete` as `delete_run`. An exact path with another method is `method_not_allowed`; all other paths are `unsupported_endpoint`. A supported request whose original path contains `?` is rejected as `query_in_path`. This is the precedence order.

For forwarded records, copy headers and body after scrubbing. Header keys are ASCII case-insensitive; replace values for `authorization`, `cookie`, `x-api-key`, `x-mlflow-token`, and `x-amz-security-token` with `[REDACTED]`. Recursively visit body objects and arrays. Replace the complete value of keys `password`, `secret`, `client_secret`, `token`, `access_token`, `api_key`, `credential`, `x-amz-credential`, `x-amz-signature`, and `x-amz-security-token`, case-insensitively. In any object with string `key` and a `value`, redact `value` when the lowercased key is `mlflow.user`, `mlflow.source.name`, `mlflow.source.git.commit`, or contains `password`, `secret`, `token`, `credential`, or `api_key`. For every HTTP or HTTPS URL string, redact query values whose keys match any sensitive header or body key, then serialize its query with Go `url.Values.Encode`. Reject a supported request as `unsafe_uri` if any body URL has userinfo, uses a scheme outside `http`, `https`, `s3`, `gs`, `wasbs`, `dbfs`, and `file`, or is a `file` URL with a host. Rejected records contain no headers, body, method, or path.

The seed must be exactly 64 lowercase hexadecimal characters. Each output line is compact JSON with HTML escaping disabled and a final newline. Before adding `chain`, encode the exact record as canonical Go JSON. A forward base record is `{"seq":integer,"decision":"forward","route":string,"request":{"method":string,"path":string,"headers":object,"body":JSON}}`; missing headers become `{}` and missing body becomes `null`. A reject base record is `{"seq":integer,"decision":"reject","reason":string}`. Set `chain = lowercase_hex(SHA256(previous_chain_bytes || canonical_base_record_bytes))`, where the first previous value is the 32 decoded seed bytes and each later value is the preceding 32-byte chain digest. Add `chain` as the final top-level field. Map keys follow Go JSON's deterministic ordering. Write the entire ledger atomically using a sibling temporary file, sync, close, and rename.
