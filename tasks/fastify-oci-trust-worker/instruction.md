# Repair the Fastify OCI trust worker

The release-admission service in `/app` accepted a forged container during an incident because its background worker followed a mutable remote tag and treated an OCI annotation as authenticated evidence. Repair the JavaScript implementation so `node /app/bin/trust-worker.js verify --request <json> --out <directory>` and the Fastify `POST /v1/admissions` endpoint evaluate the same immutable evidence and produce deterministic results.

The incident corpus originally comes from the Hugging Face dataset `terminal-bench/oci-trust-incident-corpus`; a pinned, build-time snapshot is installed at `/app/corpus` because production verification is disconnected. `/app/docs/trust-contract.md` is the authoritative contract for the snapshot schema, signed-tag verification, OCI digest binding, repository-hook policy, rejection precedence, JSON artifacts, and Graphviz graph. Preserve `GET /healthz` and the executable path `/app/bin/trust-worker.js`. The verifier supplies new corpus snapshots, bare Git remotes, keys, policies, and requests, and compares both the worker's byte-exact artifacts and `dot -Tplain` output. Do not modify `/app/corpus`, weaken the offline boundary, or encode decisions for the shipped example.
