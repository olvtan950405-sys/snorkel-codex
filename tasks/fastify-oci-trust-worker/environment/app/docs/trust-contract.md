# OCI trust worker contract

## Inputs and dataset snapshot

The worker is a pure offline verifier. A request has exactly the string fields `image`, `tag`, `platform`, `remote`, `corpus`, `policy`, and `keyring`. Values must be nonempty Unicode scalar strings. `image` is an OCI repository name (`[a-z0-9]+` components separated by `.`, `_`, `-`, or `/`); `tag` and key ids match `[A-Za-z0-9][A-Za-z0-9._-]{0,127}`; platform is `linux/amd64` or `linux/arm64`. Paths are absolute. `remote` names a bare Git repository and the other paths name a snapshot directory and JSON files. Symlinks and non-regular input files are invalid.

`records.json` is the pinned build snapshot of `terminal-bench/oci-trust-incident-corpus`. It is a JSON array of objects with exactly `image`, `tag`, `platform`, `commit`, `artifact`, `artifact_sha256`, and `artifact_size`. Strings follow the request grammar; commit is 40 lowercase hex; digest is 64 lowercase hex; size is an integer 0..1,000,000,000. Artifact is a safe relative POSIX path with no empty, `.` or `..` component. Records are unique by `(image,tag,platform)` and bytewise sorted by that tuple. There is no trusted verdict field.

The policy has exactly `policy_version`, `allowed_release_keys`, and `required_hooks`. The version is a nonempty scalar string. Release keys are unique key ids in bytewise order. Required hooks is a nonempty object mapping safe repository paths below `.githooks/` to lowercase SHA-256 values; keys are bytewise ordered. The keyring has exactly `keys`, an array bytewise sorted by `key_id`; each entry has exactly `key_id` and `public_key`, the latter an ASCII-armored OpenPGP public key. Key ids are unique, and only policy-listed ids are authorized.

Any schema, ordering, path, duplicate, missing-file, symlink, Git invocation, or Graphviz failure is an operational error: CLI exits 2 and writes no partial output; HTTP returns 400 `{"error":"invalid_request"}`.

## Verification

Resolve `refs/tags/<tag>` from the bare remote itself. Never use a working clone's cached ref, follow a symbolic ref, fetch, or contact a URL. Invoke Git with an argument vector, `--end-of-options` where supported, disabled system/global configuration, and no hooks or prompting. The ref must name an annotated tag object. Import only the selected authorized public keys into a fresh temporary `GNUPGHOME`; `git verify-tag --raw` must report a valid signature whose primary fingerprint corresponds to the selected keyring entry. The annotated tag must peel to the corpus `commit` and its tagger message must contain the exact trailer `OCI-Artifact-SHA256: <artifact_sha256>` once.

Read the artifact and every required hook from the committed tree, never from a checkout. Each must be a regular blob. Hooks must have Git mode `100755`, and their bytes must match the policy digest. The artifact bytes must have the recorded size and SHA-256. They must be canonical JSON (UTF-8; parsing then recursively sorting object keys and compact-encoding produces identical bytes plus one LF) with exactly `schemaVersion`, `mediaType`, `name`, `platform`, `config`, and `layers`. `schemaVersion` is 2, media type is `application/vnd.oci.image.manifest.v1+json`, name/platform equal the request, config is an object containing exactly string `digest` matching `sha256:<64 lowercase hex>`, and layers is a nonempty array of exact `{digest,size}` objects with valid digest and nonnegative integer size. Layer digests must be unique.

The request's matching corpus record is selected only after all documents validate. Verification reasons are collected, then ordered as: `TAG_NOT_ANNOTATED`, `TAG_SIGNATURE_INVALID`, `TAG_KEY_UNAUTHORIZED`, `TAG_COMMIT_MISMATCH`, `TAG_DIGEST_TRAILER_INVALID`, `ARTIFACT_MISSING`, `ARTIFACT_SIZE_MISMATCH`, `ARTIFACT_DIGEST_MISMATCH`, `OCI_DOCUMENT_INVALID`, `OCI_IDENTITY_MISMATCH`, `HOOK_MISSING`, `HOOK_MODE_INVALID`, `HOOK_DIGEST_MISMATCH`. Missing or malformed trust evidence is a rejection, not an operational error.

## Artifacts and API

Replace the output directory atomically; stale files must disappear. Write exactly `decision.json`, `trust.dot`, and `trust.plain`. JSON is recursive-key-sorted compact UTF-8 with one LF. `decision.json` contains exactly `artifact_sha256`, `commit`, `evidence_sha256`, `image`, `platform`, `policy_version`, `reasons`, `signer`, `status`, and `tag`; unavailable values are `null`. Status is `accepted` only with no reasons. Signer is the verified authorized key id. `evidence_sha256` hashes the canonical JSON bytes of the same object without `evidence_sha256`.

The directed Graphviz graph is named `oci_trust`, declares `graph [rankdir="LR"]` and nodes `request`, `tag`, `commit`, `artifact`, and one `hook:<path>` per policy hook. Every node has quoted `id`, `label`, `shape`, and `status` attributes in that order; ids and labels use Graphviz quoted-string escaping for backslash, quote, CR, and LF. Edges are `request -> tag`, `tag -> commit`, `commit -> artifact`, then `commit -> hook` in policy order. Accepted nodes use status `trusted`; otherwise the first failing stage and all downstream nodes use `rejected`. Statements and attributes use exactly two spaces of indentation, one statement per line, ending with `}\n`. Run `dot -Tplain trust.dot` and store its exact stdout as `trust.plain`.

CLI produces no stdout. `GET /healthz` is 200 JSON `{"status":"ok"}`. `POST /v1/admissions` accepts the same request plus an `X-Output-Directory` header, calls the same evaluator, and returns the decision with 200 when accepted or 422 when rejected. Fastify JSON serialization need not be byte-identical, but its parsed value must equal `decision.json`.
