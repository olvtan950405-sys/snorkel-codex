# Restore quorum verification for GlacierVault backup manifests

GlacierVault's offline verifier in `/app` has stopped being a trust boundary: it accepts a single signature, treats any parseable CBOR encoding as the signed representation, and ignores several catalog rules. Repair the Go code so `go build -o /app/bin/backupguard ./cmd/backupguard` produces a working command invoked as `/app/bin/backupguard verify --bundle <directory> --catalog <sqlite-file> --out <json-file>`. A bundle contains `manifest.cbor`, `signatures.json`, and the segment files named by the manifest. The manifest must be canonical CBOR as defined by `/app/docs/bundle-format.md`; signatures are Ed25519 signatures over the exact canonical manifest bytes. Stream each segment, validate its size and SHA-256 digest, and compute the specified Merkle root. Do not trust duplicate signers, fields copied into `signatures.json`, or policy values outside the selected SQLite catalog.

Reconstruct and enforce the temporal trust and quorum policy recorded in `/app/docs/policy-history/`, including later decisions that supersede earlier notes. Key activation, expiry, revocation, tenant membership, role, emergency exceptions, manifest time, and nonce replay state must all be evaluated from the catalog. Apply the rejection precedence and output contract in `/app/docs/verifier-contract.md`. On acceptance, record the nonce and accepted manifest digest in the catalog atomically; rejected runs must not mutate it, and concurrent attempts to accept the same nonce must yield at most one acceptance. All JSON output must be recursively key-sorted, compact UTF-8 with exactly one trailing newline. Keep the implementation offline, deterministic, and safe for malformed or truncated inputs.
