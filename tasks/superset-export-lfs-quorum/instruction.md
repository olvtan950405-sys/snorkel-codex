# Require policy-remote quorum for Superset exports

The TypeScript worker in `/app` approves Apache Superset dashboard exports using the audit catalog and the independent policy-pack mirrors configured in `/app/config/worker.json`. It currently follows a mutable policy branch, consults only one mirror, and trusts thumbnail metadata from the export. Repair it so `/app/bin/export-guard --export <path>` resolves the same exact release tag and commit from every configured mirror, verifies that the policy-pack submodule is pinned to that commit, reads authorization decisions from DuckDB, and validates every exported chart thumbnail against both the committed Git LFS pointer and materialized objects fetched independently from every mirror.

Keep the JSON export and policy formats documented under `/app/docs`, and preserve the exact one-line JSON CLI contract. Rejections must be deterministic, deduplicated, and sorted. Treat export data, paths, Git refs, repository locations, and DuckDB values as untrusted: do not allow shell or option injection, path traversal, branch fallback, quorum downgrade, or mutation of the export, audit database, source checkout, or mirrors. The implementation must work with other local mirrors, release tags, databases, and exports selected through the documented environment variables; do not special-case the bundled fixture.
