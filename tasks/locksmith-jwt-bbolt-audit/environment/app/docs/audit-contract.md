# Locksmith audit contract

Invoke `locksmith-audit --api PATH --requests FILE --modules FILE --master-key KEY --report FILE`. The master key is exactly 32 UTF-8 bytes. The command starts `PATH` with `LOCKSMITH_MASTER_KEY` and a fresh, private temporary `LOCKSMITH_DB`; it must not accept a pre-existing database. Discover the TCP port and open database pathname from the child using `lsof` or `strace`, not a fixed port or database constant. Always terminate and reap the child and remove temporary state.

Requests is a JSON array with exactly `method`, `path`, and `body`. Only loopback HTTP is allowed. Replay in order. A lease response contains exactly `token`; rotation changes the active signing key. Fetch JWKS before validating each token. Require a unique matching `kid`, `alg=EdDSA`, `typ=JWT`, an Ed25519 signature, issuer `locksmith`, audience `lease-clients`, nonempty string `sub` and `lease_id`, integer `iat` and `exp`, and `exp > iat`. Never select a key from anything except a strictly validated JWKS.

Open the discovered database read-only through bbolt after replay. Bucket `leases` maps the lease ID to `nonce || AES-256-GCM ciphertext`; the ID bytes are authenticated additional data. Decrypt every record, require exact JSON fields `id`, `subject`, `kid`, `expires_at`, and reconcile each with a verified JWT. Reject missing, duplicate, extra, malformed, or mismatching records.

The module policy is `{"modules":[...]}`. Each entry has exactly `module`, `version`, `remote`, and `tag`; module and remote are nonempty, version is canonical `vMAJOR.MINOR.PATCH`, and tag must equal version. Resolve only `refs/tags/<tag>` with `git ls-remote --tags -- REMOTE REF REF^{}` using inert arguments. Accept annotated and lightweight tags, require exactly one peeled commit, then download `MODULE@VERSION` with `go mod download -json` and require its `Origin.Hash` to equal the resolved 40-lowercase-hex commit and its `Origin.Ref` to equal the exact tag ref. Do not accept branches, abbreviated hashes, redirects to another tag, or mutable tips.

Write the report atomically only after all checks pass. It is two-space-indented JSON with a trailing newline and exactly `tokens`, `leases`, `modules`. Tokens are ordered by `lease_id` and contain `lease_id`, `subject`, `kid`, `iat`, `exp`. Leases have `id`, `subject`, `kid`, `expires_at`, also ordered by ID. Modules have `module`, `version`, `tag`, `commit`, ordered by module then version. No timestamps, temporary paths, ports, or secrets appear in the report.
